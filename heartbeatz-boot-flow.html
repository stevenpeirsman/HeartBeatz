HeartBeatz — Boot & Calibration Flow

Power on → auto-detect → calibrate → live sensing — fully touchscreen-driven, no keyboard needed

Complete Boot Sequence

Hardware
Power On
T+0s
Press power button on enclosure. UGREEN power bank feeds MeLE (USB-C PD) + Router (USB-A). Router boots in ~15s, MeLE in ~20s.
Automatic
Ubuntu Auto-Login
T+20s
MeLE boots Ubuntu 22.04, auto-login to heartbeatz user. No desktop environment — straight to kiosk.
systemd auto-login via getty override → ~/.bash_profile triggers kiosk script
Automatic
Docker Compose Up
T+25s
Starts sensing-server container. Binds ports 3000 (REST), 3001 (WS), 5005/udp (CSI ingest). Server starts in auto source mode — probes for ESP32 nodes.
docker compose up -d via systemd service, restart policy: unless-stopped
Automatic
Chromium Kiosk Launch
T+30s
Chromium opens in kiosk mode (fullscreen, no address bar, no cursor) on VoCore display (800×480). Navigates to http://localhost:3000/setup.
chromium --kiosk --touch-events --noerrdialogs --disable-translate --no-first-run http://localhost:3000/setup
Hardware
ESP32 Nodes Boot
T+10-25s
Nodes power on from 18650 UPS, connect to HeartBeatz WiFi (10 retries), start streaming CSI frames to 192.168.8.10:5005. Boot takes ~5-8s, WiFi connect ~3-10s.
Automatic
Setup Wizard — Splash Screen
T+30s
VoCore shows HeartBeatz logo + "Starting up..." with animated progress. System waits for Docker health check to pass.
Automatic
Setup Wizard — Node Discovery
T+35s
Server detects ESP32 nodes as UDP frames arrive on :5005. Screen updates live: each node appears as a card with signal strength (RSSI) and node_id. Waits until at least 2 nodes are detected, or 60s timeout.
REST poll: GET /api/v1/sensing/latest → check nodes[] array length and per-node RSSI
User Touch
Confirm Nodes → "Start Sensing"
T+40-60s
User sees all detected nodes, verifies count is correct. Taps "Start Sensing" button. Optionally taps "Calibrate Room" for a 30s empty-room baseline scan (v2 feature).
Automatic
Transition to Live View
T+45-65s
Navigates to simple dashboard (default on VoCore) or Observatory 3D. Swipe left/right to toggle between views. Full Observatory available on laptop at http://192.168.8.10:3000.

System Architecture — What Needs to Be Built

1. Kiosk Boot Script
Shell script + systemd service that runs on MeLE boot:
• Auto-login to heartbeatz user
• Start Docker containers (sensing-server)
• Wait for /health endpoint to respond
• Launch Chromium in kiosk mode on VoCore display
• Handle crash recovery (restart Chromium if it dies)

File: /opt/heartbeatz/kiosk.sh + heartbeatz-kiosk.service
2. Setup Wizard UI
Single-page web app served at /setup:
• Splash screen with boot progress
• Node discovery (polls /api/v1/sensing/latest)
• Node cards: ID, RSSI, status indicator
• "Start Sensing" button → redirects to /dashboard
• "Calibrate Room" button → future v2 feature

Optimized for: 800×480 touch, large tap targets (min 44px)
3. Simple Dashboard
Lightweight 2D view at /dashboard:
• Room map with node positions + person dots
• Vital signs cards (HR, BR, motion)
• Node status bar (count, signal quality)
• Swipe gesture or button to toggle → Observatory 3D

Tech: Vanilla JS + Canvas, no Three.js (keeps it fast on VoCore)
Data: WebSocket /ws/sensing for real-time updates
4. Observatory 3D (Adapted)
Existing Three.js Observatory, adapted for 800×480:
• Reduce post-processing (no bloom on VoCore, save GPU)
• Larger HUD text for small screen
• Touch controls: swipe to orbit, pinch to zoom
• Swipe right to go back to simple dashboard

Note: Full Observatory still available at :3000/observatory.html on laptop browser via WiFi

VoCore Display Setup

The VoCore 5" connects via USB-C and presents as a USB display device. Ubuntu needs DisplayLink driver or udl kernel module. Chromium targets this display via --display=:0 with Xorg or via cage/wlroots Wayland compositor.

OptionApproachComplexity
A: Xorg + xinitMinimal X server, xinit chromium on VoCore displayLow — well documented
B: Cage (Wayland)Single-app Wayland compositor, no window decorationsLow — modern, touch-friendly
C: Weston kioskWeston compositor in kiosk shell modeMedium — more features than needed

Recommendation: Option A (Xorg + xinit) for v1 — simplest, most ESP32/embedded community docs available. Switch to Cage if touch issues arise.

System Overview

HeartBeatz is a self-contained, battery-powered demo appliance that demonstrates WiFi Channel State Information (CSI) sensing. It detects human presence, pose, movement, and vital signs using only WiFi radio signals — no cameras, wearables, or contact sensors needed.

ADR-018 Binary Frame Format

Every CSI frame from ESP32 follows this exact binary layout (little-endian):

OffsetSizeFieldTypeDescription
04Magicu32 LE0xC5110001 — identifies CSI frame
41Node IDu8Sensor node identifier (0-255)
51Antennasu8Number of RX antennas (typically 1 on ESP32-S3)
62Subcarriersu16 LENumber of OFDM subcarriers (56 typical)
84Frequencyu32 LECenter frequency in MHz (e.g., 2437 for ch6)
124Sequenceu32 LEMonotonic frame counter
161RSSIi8Received signal strength (dBm)
171Noise Floori8Noise floor (dBm)
182Reserved—Zero-padded for alignment
20NI/Q Datai8[]Raw CSI: pairs of [I, Q] per subcarrier per antenna

Typical frame: 20 + (1 antenna × 56 subcarriers × 2 bytes) = 132 bytes. With 3 antennas: 356 bytes.

Additional Frame Types

0xC5110002Edge VitalsESP32 edge processingPresence, fall, breathing, HR, motion energy, person count
0xC5110004WASM OutputESP32 WASM runtimeCustom events from user-deployed WASM modules

Signal Processing Pipeline

Raw CSI I/Q data goes through 9 sequential processing stages before producing sensing results:

1
CSI Preprocessing
csi_processor.rs
Raw I/Q intake, complex number conversion, amplitude/phase extraction. Config: sampling_rate, window_size, overlap, noise_threshold.
2
Phase Sanitization
phase_sanitizer.rs
Phase unwrapping to remove 2π discontinuities from multi-path reflections. Configurable unwrapping method.
3
Hampel Filter
hampel.rs
Robust outlier removal using median + MAD (Median Absolute Deviation). Window=3, threshold=3.0σ. Returns filtered signal + outlier indices.
4
Spectrogram
spectrogram.rs
STFT time-frequency decomposition. Windows: Hann, Hamming, Blackman. Attention-gated noise suppression via RuVector min-cut graph partitioning.
5
Fresnel Zone Analysis
fresnel.rs
Physical model: ΔΦ = 4π·d/λ. Maps CSI amplitude variations to body displacement through Fresnel zone crossings. Validates breathing matches expected physical model.
6
Body Velocity Profile (BVP)
bvp.rs
STFT → frequency-to-velocity mapping (v = f_doppler × λ / 2). Scaled dot-product attention over STFT rows (RuVector). Config: window=128, hop=32, velocity_bins=64.
7
Feature Extraction
features.rs
Extracts: AmplitudeFeatures, PhaseFeatures, DopplerFeatures, PowerSpectralDensity, CorrelationFeatures (inter-antenna/subcarrier).
8
Motion Detection
motion.rs
Presence + motion scoring with configurable thresholds. Outputs: HumanDetectionResult (confidence, MotionScore), classifies absent/present_still/active.
9
Vital Signs Extraction
vital_signs.rs
FFT spectral analysis on rolling buffers. Breathing: 0.1-0.5 Hz (6-30 BPM). Heart rate: 0.67-2.0 Hz (40-120 BPM). Outputs with per-metric confidence scores.

ESP32-S3 Firmware

ESP-IDF v5.2, C, ADR-018 compliant — runs on Heemol ESP32-S3 N16R8

Boot Sequence (app_main)

The firmware initializes 12 subsystems in order. Each is fault-tolerant — failures log warnings but don't halt boot:

1
NVS Flash Init
Initialize non-volatile storage. Auto-erases on version mismatch.
2
Load NVS Config
nvs_config_load() — reads 30+ fields from NVS namespace "csi_cfg", falls back to Kconfig defaults.
3
WiFi STA Init
wifi_init_sta() — connects to HeartBeatz network, 10 retries, WPA2-PSK (or open if no password).
4
UDP Sender Init
stream_sender_init_with(target_ip, target_port) — opens UDP socket to MeLE, ENOMEM backoff 100ms.
5
CSI Collection
csi_collector_init() — enables promiscuous mode, registers CSI callback, configures LLTF/HTLTF/STBC, 50Hz rate limiter.
6
Edge Processing
edge_processing_init() — configurable tier (0-3), presence/fall thresholds, vital sign window (256 samples).
7
OTA Server
ota_update_init_ex() — HTTP server for firmware updates. Shares httpd_handle with WASM upload.
8
WASM Runtime
wasm_runtime_init() — WASM3 interpreter for user-deployed sensing modules. Periodic on_timer() callback (default 1s).
9
mmWave Sensor
mmwave_sensor_init() — auto-detect radar sensor on UART. Optional complementary sensing. Graceful skip if absent.
10
Swarm Bridge
swarm_bridge_init() — connects to Cognitum Seed (if URL configured). Heartbeat + ingest intervals.
11
Power Management
power_mgmt_init(duty_cycle) — configurable duty cycle 0-100%. Manages sleep/wake for battery life.
12
Display Task
display_task_start() — AMOLED display (if CONFIG_DISPLAY_ENABLE). Shows status on node's local display.

CSI Collector Detail

The CSI callback fires on every WiFi frame received in promiscuous mode (100-500+ times/sec). Rate limiting caps UDP output to 50Hz:

// Rate-limit: 20ms = 50Hz max send rate
#define CSI_MIN_SEND_INTERVAL_US  (20 * 1000)

// CSI config enables all LTF types for maximum subcarrier data
wifi_csi_config_t csi_config = {
    .lltf_en = true,           // Legacy Long Training Field
    .htltf_en = true,          // HT Long Training Field
    .stbc_htltf2_en = true,    // STBC HT-LTF2
    .ltf_merge_en = true,      // Merge LTF types
    .channel_filter_en = false, // Raw, unfiltered
};

// Promiscuous filter: management + data frames
filter_mask = WIFI_PROMIS_FILTER_MASK_MGMT | WIFI_PROMIS_FILTER_MASK_DATA;
